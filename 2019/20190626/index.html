Hugo搭建个人博客（二）—— 托管到Github

Written by 喵星澜 with ♥ on June 26, 2019 in

创建Github仓库（repo、repository）

  • 登录Github账号（默认已经拥有Github账号，没有就注册一个）
  • 点击右上角头像，选择Your repositories
  • 选择绿色的按钮New
  • 在Repository name填入仓库名称，一定要是yourName.github.io的形式，将yourName替换成你的Github用户名。其它什么都不要做，直接按Create repository
  • 新创建好的仓库如图所示： 注意红框中的信息，这是仓库地址，一会要用到

使用Git上传内容到仓库

  • 打开终端，转到站点根目录
cd MySite
  • 运行服务器，其中yourName自行替换为Github用户名
hugo --theme=LeaveIt --baseUrl="https://yourName.github.io/"
  • 执行完后会在MySite下创建一个名为public的文件夹，这是要上传内容的根目录
cd public
  • 执行以下内容： 
touch README.md
git init	//初始化本地仓库
git add README.md
git commit -m "first commit"
git remote add origin git@github.com:xxx/xxx.github.io.git    //将git@github.com:xxx/xxx.github.io.git替换为上图红框中内容
git add -A
git push -u origin master	//-u表示第一次提交，如果不是第一次不要加-u
  • 命令成功执行后终端会显示类似如下内容：
  • 打开Github仓库，会发现仓库变成了这样：
  • 在浏览器中访问https://meow-stars.github.io/（注意地址要替换成自己的），可以看到博客已经搭建好了

错误解决

一、

在执行git remote add origin git@github.com:xxx/xxx.github.io.git时如果报错：

fatal: remote origin already exists.

执行git remote rm origin，再重新执行git remote add origin git@github.com:xxx/xxx.github.io.git。

二、

如果不是第一次向仓库上传文件，而是要进行更新，在执行git push origin master可能会报错：

error: failed to push some refs to ‘git@github.com:xxx/xxx.git’ hint: Updates were rejected because the remote contains work that you do not have locally….

则先执行git pull origin master，再重新执行git push origin master。

更新Github

终端执行以下代码：

cd MySite
hugo --theme=LeaveIt --baseUrl="https://yourName.github.io/"
cd public
git status
git add -A
git commit -m "second commit"
git pull
git push origin master

脚本自动部署

每次更新都要重新输入这么多代码，不符合我懒宅废的处事原则，所以选择使用脚本，这样只要执行脚本就可以进行更新了。

在网站根目录MySite创建一个脚本文件deploy.sh。

注意替换掉baseUrl

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"
# Build the project.
# 替换baseUrl
hugo --theme=LeaveIt --baseUrl="https://yourName.github.io/" # if using a theme, replace with `hugo -t <YOURTHEME>`
# Go To Public folder
cd public
# Add changes to git.
git add -A
# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"
# Push source and build repos.
git pull
git push origin master
# Come Back up to the Project Root
cd ..

还没完，接下来：

  • 打开终端，转到站点根目录
cd MySite
touch deploy.sh
  • 给脚本文件可执行权限
chmod +x deploy.sh
  • 每次写完文章后只要执行脚本文件
./deploy.sh
